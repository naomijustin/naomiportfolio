tech2/iinet Customer Portal Front End
Overview of Project
Existing tech2/iinet customer web portal was due for a redesign and modernisation. Approach was to upgrade the look and feel, add elements that would modernise the portal to current trends (ie. responsiveness), while maintaining usability for the existing customer-base.

UX/UI
I created the designs to echo the layout of the previous portal - placing elements in categories remained the same, however had a cleaner look and allowed to compact the fields within the viewport while retaining clarity and legibility.

I created multiple designs in Procreate to present to the business. Ultimately, the business chose the design that had elements that were similar to the original portal (ie. card titles had similar bordering background, layout was in a similar location) however this was closer to a “dark-mode” version, as well as being more modernised and on-brand.

Chosen Design


Existing Portal that needed the redesign


Front End Development
I built the front end of the portal locally in Visual Studio Code, creating test JSON data to ensure that Ajax (POST and GET) methods were working before giving files over to the Back End Developers to connect to the API on the server.

The “Login” page was built with CSS Grid, Bootstrap, JavaScript and Ajax where the credentials are validated through the API on the server.



Once logged in, the user is then directed to the index page, or the blank form that allows a user to create a new ticket. There is also a prompt to allow the user to save their credentials in Chrome (triggered by the POST method in Ajax).



This ticket booking form has multiple dynamic dropdowns determined by parent selections, a “filter as you type” suburb dropdown, a file uploader and a calendar appointment selection. Technologies used here include the Full Calendar Plugin, Bootstrap, JavaScript, jQuery, Ajax and CSS grid. In the header, there are also “wave” SVGs that preserve the height and do not preserve the aspect ratio when resizing the browser to keep a sense of organic flow to the shapes without disrupting the lower section layout of the page.





Within this index page, other than booking a new ticket, the user also has the option of looking up existing tickets in the database - also pulled dynamically via Ajax from the JSON data file (using test data).

If the user chooses an existing open, cancelled, or completed ticket, this opens a modal that contains a log of actions or comments made on this ticket - pulled dynamically via Ajax. There is the ability to download the existing data viewed here as multiple file formats (Word, Excel, HTML etc.).



Depending on the status of the ticket in question, there are also dynamic buttons such as a cancelation option, and a View Documents option (if documents exist for this ticket). The View Documents section brings the user to a secondary modal where the user has the option of downloading a selection of PDF file, while still retaining the ability to go back to the current ticket view as to not disrupt the flow of the User Experience and prevent the user from needed to search the same ticket again.



Accessibility
All elements in the portal have a “tab-index” and can be accessed only via keyboard or only via mouse selections. Focus was also customised to keep the user aware of what would be selected while keeping on brand and aesthetically pleasing. All images have an alternative text to allow any broken links to still allow the user to not miss any information. And finally, all text has been kept above 12pt, and responsiveness worked correctly down to 320px browser width as these are both minimum required across web standards.

Haberfield Baptist Church Website and Youtube
Overview
Redesigned, built and maintained Wordpress Site with Genesis Framework with Dynamik Theme. Set up hosting, domain pointing, SEO, Google Analytics, Calendar Events, along with Ad Hoc tasks and tech support.
Filmed, synced, edited and uploaded weekly sermons 2015 - mid 2020.

Rebuild the site from scratch due to issues with access of the current outsourced host and setup.
Requirements were to allow the client to make their own changes without contacting an admin, while also keeping within a modest budget. Multiple users needed to have access to the site to make changes (in certain areas) and to also set up 30 emails with unlimited server space.

Proposed and implemented solution
Of a few solutions, building in Wordpress hosted through Hostgator satisfied the majority of the client's needs. Due to the key priority at the time of building the site being their modest budget, this was the chosen solution.

This solution also allowed the client to have unlimited custom emails with unlimited server space, along with multiple Editor accounts that the client could log into without needing access to the Admin account.

Logistics
The theme chosen was the Dynamik Theme using the Genesis Framework due to it’s customizability and relative low cost, while still maintaining a professional look to the site.

Landing Page


I worked closely with the Associate Pastor, creating their website to their liking. I implemented SEO by Yoast and Google Analytics to provide the client with information about the site’s traffic and which pages were performing well.

Vision Page


I also maintained their site between 2015 - 2020, providing tech support for emails and ad hoc tasks, updating the website with elements such as pop ups, embedding Google Calendar, updating embedded Youtube Links for their sermons and providing advice for future projects (updating film equipment, logistics for a “members only” section on the website, etc.).

Film & Editing
I filmed the sermons each week between 2015 and 2020, starting on an older camera and burning to DVDs, but eventually upgrading to a Panasonic HC-VX980M, and recording sound separately with a Zoom H4 that records both atmosphere, as well as the microphone on a separate track (via plugging in an XLR to the Zoom).

From there, I would then sync the Zoom and Footage (removing the footage audio - as only there for syncing purposes) and then edit the sermon (in Adobe Premiere Pro) usually ranging between 45 min - 1.5 hrs. Once edited and cleaned up sound with title cards, overlays of readings, images and outro cards, I would then export this and upload the file to Youtube in 1080p 25 (H.264).

Once in Youtube, I would add additional outro links for any viewers to click on (ie. the Channel, the most recent video and a recommended video from the channel).

I would also go through the video and find a shot of the speaker that was flattering (and dynamic with other Thumbnails on the channel) and create a Thumbnail for the relevant video in Procreate on the iPad Pro, on a background of the clients logo.

I'm The Boss Game
Game Description
You're the boss of a failing company, who has to take on multiple responsibilities just to keep the business afloat, but the stress is getting to you. Keep on top of daily tasks before the business collapses.


Overview of Creation
Created in the January 2020 (48 hour) Global Game Jam at the AIE (Academy of Interactive Entertainment) Jam Site, prompt give was "Repair".

We worked in a team of 2, I was the Game Artist and my colleague was the Game Programmer. We jointly worked on the design of the playability of the game.

We were inspired by the retro, mundane atmosphere and colour palette of the 1999 film “Officespace”, which parodies corporate office culture.

Logistics
We decided to experiment and make a 2D game creating the illusion of faux 3D Space, in order to create a sense of a maze of corridors and cubicles, where the only landmarks are water coolers in the office. All assets I created were 2D, and depending on the camera angle would either change at a 90 degree angle or always be facing the camera. I also created and animated a short opening sequence to give some context to the player.



Artistic Design
All assets were made using Aseprite via Astropad (Screen Mirroring) through the iPad Pro.

For colours, I decided to stay true to the genre and create a very bland, grey/brown theme, the only splash of colour being the occasional tie, suspenders, computer screen or the bosses expensive classic Casio watch.

We kept the lighting quite dark as to feel a bit claustrophobic, and to create a more challenging navigation around the office for the player.

For atmosphere, I found a great 16-bit style soundtrack and sourced copyright free in sounds of typing, phones ringing, as well as us recording and adding in the occasional grumble from the main character about a memo or an overdue TPS Report.





I designed the health bar or “Sweat Meter'' down the bottom left corner of the screen to be different from a typical bar that fills up. We wanted to not give the player too much information to create some suspense on if they will get to the next task in time or not, so we opted for 25% increments. I created the boss's face to slowly be drowning in sweat, filling up ends the game. I felt it was important to add a unique humorous aspect that is typically just a graphical UI element, but I feel due to it always being visible and checked consistently by the player, add some charm and humour to the game.


Final Thoughts
All in all, we were happy with the result we created in such a short amount of time and being in a team of 2.

Overview of Creation
Created in the August/September 2019 (30 hour) Viva League Game Jam, prompt given was “Secret”.

We worked in a team of 2, I was the Game Artist and my colleague was the Game Programmer. We jointly worked on the design of the playability of the game.

We decided to interpret the prompt as uncovering a ‘secret’ and were inspired to create a detective game. We also liked the play of words of “Billy” and “Kid” as they both alluded to goats and incorporated them into a detective goat character in a Western style environment.



Logistics
We decided to create a 2D side scrolling shooter, however really loved the idea of creating the illusion of depth so created a multiple parallax effect. This was achieved by creating multiple background layers (different variations that could be stitched together seamlessly in order to create a more unique look rather than an obvious repeating background) and have each layer scroll at a different rate as the character walks.

We made the game quite difficult to beat as we wanted it to be challenging. Due to this, we added in a “Bulletproof” Toggle (up the top right corner) so that players would be able to get through the game. However, in order to not make it too easy, we added that if the main character got too close to an enemy, the enemy would pull out a stick of dynamite and self destruct, potentially killing the main character in the process.

To tell the story and give the player context, I created art comic panels at the beginning and the end of the game to tell the story.





Artistic Design
All in-game assets were made using Aseprite via Astropad (Screen Mirroring) through the iPad Pro. Title Cards and Comic Strips were made with Procreate on the iPad Pro.

For colours, I decided to use a very warm orange/red/brown palette, contrasted with a vibrant blue - typically seen in colour-grading in cinema, as well as very commonly found in Western films.

For atmosphere, I was inspired by films such as “The Good, The Bad and The Ugly” and “Once Upon a Time In The West” - soundtracks both created by Ennio Morricone.
I used 2 different copyright free soundtracks for the opening and in game that alluded to a deep electric guitar and wind blowing that creates the feel of a high noon duel.
Final music I found - used at the end of the game is an old-timey piano sound that is usually played in a saloon.



Awards
The August/September 2019 30 hour Viva League Game Jam was part of a competition for winning $500 for Best Game Overall. There were about 10 - 15 games submitted in total by other teams. Through an anonymous vote, our game won first place for “Best Game Overall”.

Just Us Game
Game Description
A lady and her child fight against all odds to survive. Home is not a location but a uniting force.

Overview of Creation
Created in the January 2019 (48 hour) Global Game Jam, the prompt given was “What home means to you”.
We worked in a team of 6, with 3 artists, 2 programmers and 1 sound designer. I was the Background Artist and created repeating in-game background elements that could be placed by the programmers into the game canvas, along with creating the credits title card.



Logistics
We created a 2D side scrolling puzzle game, however due to the size of the team and the new use of Unity as a tool (instead of previous familiar Game Maker) there were challenges in creating elements (such as the opening title card) and importing elements that were created but didn’t make it into the final delivered game.
Instead, due to limited knowledge of this engine, we opted to focus our priorities on playability rather than look in order to meet the submission deadline.





Artistic Design
Due to having 3 different artists on the team, we decided to go with an 8-bit pixel art style - partially due to the beautiful aesthetic, but partially due also to less variation in the art style so that there would be more consistency throughout the game’s theme.

We chose to lower the saturation of all colours, bar the main character to create a pastel yet gloomy environment, in order to also make the main character stand out.



Design and Mockup
All Background elements were separated to give more variety. The footpath and sky remained the same and I designed them to be able to stitch together seamlessly, and each building and plant was created to be placed into the scene. To again give further variation to the scene, once each building created, I adjusted and exported different Hues of that element.

Once all elements were completed, I created a mockup of the background to give to the programmers so they had a guide on how I designed the elements tot be placed into the canvas.



Final Thoughts
While we were satisfied with the results, a smaller team with a singular vision and being very confident with the engine chosen we felt would make a stronger game in future jams.

Kaitalin's Curse Game
Game Description
Kaitalin, an Eastern European grandmother, is cursed and turned into a mouse when she tries to steal a gypsy-witch’s luxurious bracelet. Escape the curse in this mystic and spooky action platformer.



Overview of Creation
Created in the January 2016 (48 hour) Global Game Jam, the prompt given was “Ritual”.
We worked in a team of 4, with 1 artist, 2 programmers and 1 sound designer. I was the Game Artist and created all the assets, backgrounds, title cards and animations.

Logistics
The game revolves around the grandmother-turned-mouse retrieving beads of the broken gypsies bracelet in order to undo the spell. The player has to navigate through parts of the walls, only lit by a candle light that is slowly running out. The player has to avoid spiders, navigate through different sections of the wall, and make their way back in time with a bracelet bead to the gypsie in order to refill their candle (or time) to look for the remaining beads.

While this game is technically a platformer, it also lends itself to being described as a dungeon crawler as each section of the wall that the mouse walks into is it’s own “room”. The camera doesn’t pan with the player, but is rather fixed and switches to a new room, each time the player leaves the previous room. Part of the designed difficulty of this game is the navigation due to obstacles the player has to avoid in an ever diminishing lighting and time setting.





Artistic Design
I kept the colour palette to a warm tone, with shades leaning towards purples, reds, oranges, yellows and browns. Colours in this game were kept vibrant and bold to keep to the Eastern European theme.
At the time, my process was sketching out my ideas in a sketchbook, taking a photo and then using this as a guide in Photoshop and digitally creating that image via a connected Wacom Tablet. This was a very lengthy process so I kept the enemies in the game to spiders only (however did animate) to add a creepy-crawly feel. I decided to go with dark line art, and shade with a flat base, shadow and highlights.

Due to the lengthened time of this process, as time ran out, I created a basic comic panel at the beginning of the game to create a sense of context before the player dove into the game.
